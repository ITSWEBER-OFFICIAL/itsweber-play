# Whisper Auto-Captions

Automatic subtitle generation via [whisper.cpp](https://github.com/ggerganov/whisper.cpp). Runs as a separate BullMQ queue after transcoding completes.

## Enabling the feature

```yaml
# docker-compose.yml — worker service
environment:
  AUTO_CAPTIONS_ENABLED: "1"
  WHISPER_MODEL: /models/ggml-small.bin
volumes:
  - models:/models
```

The model is **not** baked into the Docker image (500 MB). Download it once into the volume:

```bash
docker compose run --rm worker \
  wget -P /models \
    https://huggingface.co/ggerganov/whisper.cpp/resolve/main/ggml-small.bin
```

## How it works

1. Transcode job finishes → video is `LIVE`.
2. If `AUTO_CAPTIONS_ENABLED=1`, a `caption-generate` job is enqueued.
3. Caption worker downloads the first 6 HLS segments from MinIO.
4. FFmpeg concatenates segments into a 16 kHz mono WAV.
5. `whisper-cli` runs with `--output-vtt`, producing a `.vtt` file.
6. VTT is uploaded to `play-assets/captions/<videoId>/auto-<lang>.vtt`.
7. A `VideoCaption` row is created with `isAutoGenerated: true`.

## Docker build

`whisper-cli` is compiled from source in a multi-stage build (`whisper-build` stage, Debian Bookworm). The binary is copied into the runtime image. No GPU — CPU-only inference.

```dockerfile
ARG WHISPER_VERSION=1.7.3
FROM debian:bookworm-slim AS whisper-build
# … cmake build …
COPY --from=whisper-build /usr/local/bin/whisper-cli /usr/local/bin/whisper-cli
```

## Environment variables

| Variable | Default | Purpose |
|---|---|---|
| `AUTO_CAPTIONS_ENABLED` | unset | Set to `"1"` to enable |
| `WHISPER_PATH` | `/usr/local/bin/whisper-cli` | Path to whisper binary |
| `WHISPER_MODEL` | `/models/ggml-small.bin` | Path to GGML model file |

## Performance

- `ggml-small.bin` (~244 MB) is a good speed/quality tradeoff on CPU.
- Caption worker concurrency is fixed at `1` — Whisper is CPU-heavy.
- A 90s video typically takes 2–5 minutes on a modern CPU core.
- For faster inference, swap to `ggml-base.bin` (~142 MB, lower accuracy).

## Disabling / opting out

Simply do not set `AUTO_CAPTIONS_ENABLED=1`. Existing auto-captions can be deleted from the Studio edit page (Captions tab → trash icon).

## Studio UI

The Captions panel in `/studio/<id>/edit` shows an **Auto** badge next to whisper-generated captions. Users can delete them and upload manual VTT/SRT files instead.
